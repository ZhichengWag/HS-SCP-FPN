# HS-FPN 低频语义分支设计文档

## 1. 原始 HFP 是如何生效的

HS-FPN 里当前的 HFP 不是一个单纯的“特征变换层”，而是一个**高频响应生成器 + 门控增强器**。它的核心作用不是直接输出新的语义，而是先从输入特征中提取“更像小目标证据”的响应，再用这个响应去重标定原特征。

对应代码主要在 `hs_fpn.py` 的 `27-140` 行，`HS_FPN` 的接入位置在 `279-325` 行，核心路径是：

- [`DctSpatialInteraction`](./hs_fpn.py) 负责空间门控
- [`DctChannelInteraction`](./hs_fpn.py) 负责通道门控
- [`HFP`](./hs_fpn.py) 把两条路径的结果相加后再卷积融合

### 1.1 空间路径的工作方式

空间路径在 `isdct=True` 时会做 DCT：

1. 对输入特征 `x` 做二维 DCT。
2. 构造一个固定掩码，把左上角低频区域置零。
3. 将 DCT 结果与掩码逐点相乘。
4. 再通过 IDCT 回到空间域。
5. 将这个回到空间域的结果作为空间注意力/门控图，与原特征逐点相乘。

可以理解为：

`空间路径输出 = 原特征 × 高频空间响应`

它的本质是：**把平滑、低变化率、容易淹没小目标的背景部分压低，把边缘和局部突变部分保留下来。**

当 `isdct=False` 时，代码不再做 DCT，而是直接用 `1x1 Conv + Sigmoid` 生成一个空间门控图，这说明作者本身已经把 HFP 分成了“频域增强”和“轻量 gating”两种模式。

### 1.2 通道路径的工作方式

通道路径在 `isdct=True` 时同样先做 DCT 高频提取，然后：

1. 对高频重建特征做 `Adaptive Max Pool` 和 `Adaptive Avg Pool`。
2. 将池化后的响应压缩为通道描述子。
3. 通过两个 `1x1 Conv` 生成通道权重。
4. 用 `Sigmoid` 输出通道门控向量，再乘回原特征。

这一路的作用是：**不是只看“哪里重要”，还要看“哪个通道重要”**。  
对小目标来说，这能更好地放大稀疏但关键的通道激活，抑制冗余背景通道。

### 1.3 HFP 的整体生效机制

在 `HFP.forward()` 中，空间路径和通道路径分别输出增强后的特征，然后相加，再经过一个 `3x3 Conv + GroupNorm` 做二次整合：

`HFP(x) = Conv3x3( Spatial(x) + Channel(x) )`

所以 HFP 的真实作用可以总结为三步：

1. **高频提纯**：把平滑背景和低变化部分压下去。
2. **双视角门控**：从空间和通道两个维度找小目标证据。
3. **局部重整**：用卷积把增强结果重新组织成可用于 FPN 传递的输出。

### 1.4 原始 HFP 解决了什么问题

它主要解决的是“小目标信号弱”的问题，而不是“语义理解不足”的问题。

更具体地说，原始 HFP 让网络更容易学到：

- 小目标边缘
- 纹理突变
- 局部对比度
- 细粒度激活

但它并没有显式建模：

- 场景布局
- 背景结构
- 目标可能出现的区域先验
- 语义上下文约束

这就是你新分支可以切入的地方。

## 2. 你要设计的分支，应该是什么

你现在想加的不是“频域低通分支”，而是一个**基于普通神经网络低频偏置的语义上下文分支**。

注意这里的“低频”不是 DCT 里的低频分量，而是指 CNN / 常规神经网络更容易先学到的：

- 粗粒度结构
- 平滑语义区域
- 大范围上下文
- 背景布局规律

因此，建议把它命名为：

- `Semantic Context Branch`
- 或 `Scene Prior Branch`
- 或 `Background-Aware Semantic Branch`

如果论文里仍然想保留“低频”说法，需要明确解释：

> 此处的低频并非显式频域低通，而是指神经网络天然更偏向学习平滑、粗粒度、全局结构的表示。

## 3. 新分支的目标

这个分支的目标不是提升边缘，而是回答两个问题：

1. 当前区域像不像一个合理的目标出现区域？
2. 当前场景背景是否支持某类小目标出现？

换句话说，它学的不是“目标是什么”，而是“**目标更可能在哪里**”。

这对小目标很重要，因为小目标本身可见性弱，单纯靠纹理很容易受背景干扰；但如果结合场景先验，模型会更容易做出位置判断。

## 4. 新分支的推荐结构

建议把它设计成一个非常轻量的 `Semantic Context Encoder`，结构尽量偏“粗语义”而不是“精分割”：

### 4.1 基本结构

输入特征 `X` 后：

1. `1x1 Conv` 降维
2. `3x3 Conv` 或 `Depthwise 3x3 Conv` 提取局部上下文
3. `Stride=2` 下采样一次或两次，扩大感受野
4. 少量残差块或轻量注意力块
5. `Global Average Pooling` 或 `Context Aggregation`
6. `Upsample` 回原分辨率
7. `Sigmoid` 输出语义引导图 `A_sem`

可写成：

`A_sem = Sigmoid(Up(ContextEncoder(X)))`

然后用它去调制原特征：

`X_sem = X ⊙ A_sem`

### 4.2 为什么要这样设计

因为你要的是“轻量、粗粒度、场景级”的信息，而不是另一个重型分割头。

如果结构过深、过复杂，它会变成一个新的检测分支，失去你原本想要的“低成本语义先验”这个特点。

### 4.3 推荐的模块配置

比较稳的配置是：

- `1x1 Conv` 压缩通道
- `DWConv 3x3` 或普通 `3x3 Conv`
- `GN + ReLU`
- `1~2` 个残差块
- `SE/GC` 风格全局上下文汇聚
- `1x1 Conv` 输出单通道或少通道语义图
- `Sigmoid` 门控

如果你希望更轻，可以只保留：

- `1x1 Conv`
- `Stride=2 Conv`
- `Global Pool`
- `1x1 Conv`
- `Sigmoid`

## 5. 这个分支如何和原始 HFP 结合

最推荐的结合方式是：**高频分支负责细节增强，语义分支负责区域引导，两者并行后融合。**

### 5.1 并行双分支

输入特征 `X` 同时进入两个分支：

- 高频分支 `F_high`
- 语义分支 `F_sem`

其中：

`F_high = HFP(X)`  
`F_sem = SemanticBranch(X)`

然后融合：

`Y = Conv([F_high, F_sem])`

这是最通用、最稳定的方式。

### 5.2 门控式融合

如果你希望语义分支不要过强地改变特征分布，可以让它只作为门控：

`Y = F_high ⊙ (1 + A_sem)`

这种写法的好处是：

- 高频细节仍然是主干
- 语义分支只做“放大/抑制”
- 不容易把语义信息注入得过猛

### 5.3 残差式融合

如果你更想保持原 HFP 的行为不变，可以做残差叠加：

`Y = HFP(X) + β · SemanticBranch(X)`

其中 `β` 可以是可学习标量，初值设小一点，比如 `0.1`。

这种方式的优点是：

- 能最大限度保留原模型收益
- 新分支是增量式的
- 更适合做 ablation

### 5.4 最推荐的最终形式

如果目标是论文里既容易讲清楚，又容易落地，我建议：

`Y = Conv([HFP(X), X ⊙ A_sem])`

这是最平衡的方案，因为：

- HFP 负责“看见目标”
- `A_sem` 负责“告诉网络哪里更值得看”
- 最终卷积负责融合两种互补信息

## 6. SAM 在这里怎么用

你的想法里提到借助 SAM，这里最重要的一点是：

**SAM 只用于训练监督，不参与推理。**

否则整个模块就不再轻量了。

### 6.1 SAM 的作用定位

SAM 不直接提供检测类别语义，但它非常适合提供：

- 区域划分
- 粗边界
- 场景结构
- 目标/背景分离

这正好适合监督语义上下文分支学习“场景布局”和“区域先验”。

### 6.2 两种可行监督方式

#### 方式 A：Box Prompt 生成目标区域

对每个 GT box：

1. 用 box 作为 prompt 喂给 SAM
2. 得到目标区域 mask
3. 将 mask 下采样到 FPN 对应层
4. 监督语义分支输出与该区域一致的引导图

这种方式的好处是：

- 标签和检测任务最一致
- 不需要额外人工标注

缺点是：

- SAM 的 mask 边界有时过细或过粗
- 目标很小时，mask 可能不稳定

#### 方式 B：背景先验监督

先让 SAM 给出目标区域，再取其补集作为背景区域：

`M_bg = 1 - M_obj`

语义分支学习的就是：

- 背景大结构
- 场景布局
- 目标更可能出现的背景区域

这种更贴合你现在的想法，因为你强调的是“背景语义信息”。

### 6.3 伪标签处理

为了让监督更稳，建议对 SAM 结果做以下处理：

- 下采样到特征层尺度
- 边界区域做 ignore 或 erosion
- 将 hard mask 转成 soft mask
- 可选 Gaussian smoothing

原因是语义分支不追求像素级精确边界，而是追求“哪里更合理”的粗粒度结构。

## 7. 训练损失怎么设计

建议总损失写成：

`L = L_det + λ1 L_sem + λ2 L_cons + λ3 L_reg`

### 7.1 `L_det`

检测主任务损失，保持原来不变。

### 7.2 `L_sem`

语义分支监督损失，用 SAM 伪标签监督 `A_sem`。

推荐：

- `BCE Loss`
- `Dice Loss`
- 或 `BCE + Dice`

如果类别不均衡严重，`Dice` 会更稳。

### 7.3 `L_cons`

高频分支和语义分支的互补约束。

目的不是让它们相同，而是避免两者学到完全重复的东西。

可以有两种写法：

- 正交约束：鼓励二者在通道或空间上差异化
- 去相关约束：降低 feature correlation

这项不是必须，但做了通常有助于提升“互补性”这个叙述。

### 7.4 `L_reg`

语义分支平滑正则项，可选。

例如：

- `TV Loss`
- 输出图平滑约束
- 低分辨率监督

它的作用是让 `A_sem` 更符合“低频偏置”的特征，而不是产生高频噪点。

## 8. 放在哪些层最合适

结合 HS-FPN 当前结构，建议不要所有层一刀切，而是分层设计。

### 8.1 高层更适合语义分支

在 `p4 / p3`：

- 语义上下文更明显
- 场景布局信息更强
- 低频偏置更容易收敛

所以这两层可以给语义分支更高权重。

### 8.2 低层更适合高频分支

在 `p2 / p1`：

- 小目标细节更关键
- 纹理、边缘、局部对比更重要

所以这里应该保持高频分支主导，语义分支只做轻量辅助。

### 8.3 推荐层级策略

一个比较合理的配置是：

- `p4`: 高频 + 语义都保留，但语义更强
- `p3`: 高频 + 语义平衡
- `p2`: 高频主导，语义轻量
- `p1`: 高频主导，语义最轻

## 9. 如何接入现有 HS-FPN

当前代码结构在 [`hs_fpn.py`](/e:/HS-FPN-main/hs_fpn.py) 中是：

- `SelfAttn_p4 = HFP(...)`
- `SelfAttn_p3 = HFP(...)`
- `SelfAttn_p2 = HFP(...)`
- `SelfAttn_p1 = HFP(...)`

并且在 forward 中：

- 先做 lateral conv
- 再做逐层 HFP
- 再做 `SDP` 跨层交互
- 最后 FPN top-down 融合

### 9.1 最小侵入式接法

最简单的做法不是改掉原来的 HFP，而是新增一个并行模块：

- `HighBranch = HFP(...)`
- `SemBranch = SemanticContextBranch(...)`

然后在每一层做：

`laterals[i] = Fusion(HighBranch(laterals[i]), SemBranch(laterals[i]))`

这样好处是：

- 原模块不动，方便做对比实验
- 风险最小
- 可以逐层加开关控制

### 9.2 更推荐的实现顺序

建议先只在 `p3/p4` 加语义分支，等效果稳定后再扩展到 `p2`。

原因：

- 语义分支太早介入，容易和底层边缘特征冲突
- 高层特征更适合学场景先验

### 9.3 一个推荐的结构草图

对于每一层特征 `F_i`：

1. `F_i^h = HFP(F_i)`
2. `A_i^sem = SemanticBranch(F_i)`
3. `F_i^sem = F_i ⊙ A_i^sem`
4. `F_i' = Conv([F_i^h, F_i^sem])`

然后把 `F_i'` 送入原来的 `SDP` 和 top-down FPN 流程。

也就是说，**语义分支最好放在 HFP 之后、SDP 之前**。  
这样做的原因是：

- HFP 先把细节增强出来
- 语义分支再告诉网络这些细节应该在哪些场景里更可信
- SDP 再做跨层空间依赖补偿

这个顺序最符合“细节 -> 语义 -> 跨层关系”的逻辑。

## 10. 推荐的最终文档化表述

你可以把方法概括为：

> 我们在原始 HFP 的高频增强路径之外，进一步引入一个基于 CNN 低频偏置的语义上下文分支。该分支不依赖显式频域低通，而是通过轻量神经网络自发学习场景布局、背景结构和区域先验，并在训练阶段借助 SAM 生成的伪分割区域进行粗粒度监督。最终，模型在高频细节增强与低频语义引导之间建立互补关系，从而提升小目标在复杂背景中的可检出性。

## 11. 实施优先级建议

如果你要把它真正落地，建议按这个顺序做：

1. 先保留原 HFP 不动，新增一个轻量语义分支。
2. 先只接到 `p3/p4`，看 mAP 和召回是否提升。
3. 先用最简单的 `BCE + Dice` 做监督。
4. 再加互补约束和分层权重。
5. 最后再考虑是否扩展到 `p2`。

这样风险最小，也最容易做出稳定的消融实验。

## 12. 最终建议

如果你想把这个方向做成论文创新点，最稳的定位不是“再做一个分割头”，而是：

- 原始 HFP 解决“看清楚”
- 新语义分支解决“看懂场景”
- SAM 只作为训练期教师，提供区域先验
- 推理阶段保持轻量，不引入额外大模型成本

这会让你的方法逻辑非常完整，也和 HS-FPN 现有“高频增强 + 空间依赖建模”的主线兼容。
